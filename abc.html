4.1

Introduction to EM Algorithm

The core problem EM solves: You have data with missing or hidden information, and you can't solve for the parameters without knowing the hidden data — and you can't infer the hidden data without the parameters. EM breaks this circular dependency by alternating between the two.

Example: The Two Coins Problem. Two biased coins exist — Coin A and Coin B with different head probabilities. Someone secretly picks one and flips it 10 times. You see the results but never learn which coin was used. EM figures out both the coin biases AND which coin was likely used.

4.2

The E-Step

E-Step (Expectation): Given current parameter estimates, compute the expected value of the hidden/latent variables. Each data point gets a soft probability of belonging to each cluster — not a hard assignment. This is what separates EM from K-Means.

Example: Heights without gender labels. We have 20 people's heights but no gender information. The E-step asks: given our current Gaussian model, how likely is each person to be from group A vs group B?

4.3

The M-Step

M-Step (Maximization): Using the soft assignments from the E-step as weights, re-estimate the model parameters to maximize the expected log-likelihood. Each data point contributes to each group proportionally to its membership probability.

Watch the full EM loop. Two Gaussian curves start at random positions. Each iteration: E-step computes soft assignments → M-step moves the means to the weighted center of each cluster. The curves chase the data.

4.4

Convergence of EM

Convergence guarantee: Every EM iteration either increases the log-likelihood or keeps it exactly the same — it can never decrease. EM always reaches a peak, but not necessarily the global maximum. Different starting points may converge to different local optima.

Try multiple random starts. Hit "New Random Start" to initialize EM at a different point. Some starts find higher optima than others — this is why real implementations run EM many times and pick the best result.

4.5

Jensen's Inequality & Lower Bound

Jensen's Inequality: For a concave function f (like log), f(E[X]) ≥ E[f(X)]. The function evaluated at the average is always ≥ the average of the function values. The chord between two points on a concave curve always lies below the curve.

Why EM never decreases. The true log-likelihood is hard to optimize directly (log of a sum). Jensen lets EM construct a simpler lower bound — always ≤ the real thing. Each iteration maximizes this bound, which provably lifts the real likelihood upward.

4.6

Gaussian Mixture Model

GMM Explainer: A Gaussian Mixture Model (GMM) represents a population as a sum of multiple Gaussian distributions. In this example, we model height as a mixture of two groups (e.g., Male and Female). EM is the standard algorithm used to estimate the means, variances, and mixing weights (proportions).
